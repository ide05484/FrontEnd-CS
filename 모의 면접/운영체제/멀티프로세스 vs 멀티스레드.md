  <summary><strong>멀티프로세스 vs 멀티스레드</strong></summary>

<br>

## 멀티프로세스
#### 1. 독립된 프로세스를 여러 개 실행
- 각 프로세스는 독립된 메모리 공간 보유
- 한 프로세스 문제가 다른 프로세스에 영향 없음

#### 2. 단점
- Context Switching 비용 큼
- 프로세스 간 데이터 공유 어려움 → IPC 필요

---

## 멀티스레드
#### 1. 하나의 프로세스 내부에 여러 실행 흐름
- Code, Data, Heap 공유 / Stack만 독립
- 빠른 컨텍스트 스위칭

#### 2. 단점
- 한 스레드 오류 시 같은 프로세스 내 전체 영향
- 동기화 문제 발생 가능

---

## 멀티프로세스와 멀티스레드 차이

| 항목 | 멀티프로세스 | 멀티스레드 |
|---|---|---|
| 안정성 | 높음 | 낮음 |
| 메모리 | 독립 | 공유 |
| 데이터 공유 | 어렵다 (IPC) | 쉽다 |
| 속도 | 상대적으로 느림 | 빠름 |
| 장애 영향 | 분리 | 스레드 전체 영향 |
