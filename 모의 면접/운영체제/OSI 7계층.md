## OSI 7 계층 개념이란?
네트워크 통신 과정을 7단계로 나눈 모델

---

## 계층 정리

| 계층 | 이름 | 역할 | 예시 |
|---|---|---|---|
| 7 | 응용 | 사용자 서비스 | HTTP, FTP |
| 6 | 표현 | 인코딩/암호화 | SSL, JPEG |
| 5 | 세션| 연결 유지/관리 | TLS, API 세션 |
| 4 | 전송 | 신뢰성·전송 제어 | TCP, UDP |
| 3 | 네트워크 | 경로 선택/라우팅 | IP |
| 2 | 데이터 링크 | 오류 제어/프레임 | MAC, Ethernet |
| 1 | 물리 | 전기 신호 전달 | 케이블, NIC |

---

## 상세 설명
#### 1계층 - 물리계층
- 1계층에서는 주로 전기적, 기계적, 기능적인 특성을 이용해서 통신 케이블로 데이터를 전송한다
- 에러 체크 X

#### 2계층 - 데이터 링크 계층
- 물리계층에서 발생할 수 있는 오류 검출·수정 및 프레임 단위 전송을 담당한다  
- MAC 주소 기반으로 같은 네트워크 내 장치 간 통신 수행

#### 3계층 - 네트워크 계층
- 목적지까지 최적 경로를 결정하고 패킷을 전달하는 역할(라우팅)  
- IP 주소 기반으로 서로 다른 네트워크 간 통신 수행

#### 4계층 - 전송 계층
- 송신·수신 프로세스 간 신뢰성 있는 데이터 전송 담당  
- TCP, UDP 사용, 포트번호 기반 통신

#### 5계층 - 세션 계층
- 통신 세션을 생성/유지/종료하고 데이터 교환 동기화 관리  
- 연결 관리, 인증, 재전송 제어 등 수행

#### 6계층 - 표현 계층
- 데이터 인코딩/디코딩, 압축, 암호화 수행하여 애플리케이션이 이해할 수 있는 형태로 변환  
- SSL/TLS, JPEG, MPEG 등 포맷/보안 처리

#### 7계층 - 응용 계층
- 사용자와 직접 상호작용하는 계층으로 애플리케이션 서비스 제공  
- 웹·메일·파일 전송 등 사용자가 체감하는 서비스 (HTTP, FTP, SMTP)
