## Code 영역
#### 1. 기계어 코드 저장 영역
- 실행할 프로그르매의 코드가 저장됨
- 읽기 전용 메모리

#### 2. 공유 가능
- 여러 프로세스가 같은 코드를 공유할 수 있음

---

## Data 영역
#### 1. 전역 변수/정적 변수 저장
- 프로그램 시작 시 할당되어 종료까지 유지됨
- 전역 변수, static 변수가 저장됨

#### 2. 초기화 여부에 따름
- 초기화된 데이터 → Data 영역
- 초기화되지 않은 데이터 → BSS 영역

---

## Stack 영역
#### 1. 함수 호출 시 사용되는 메모리
- 지역 변수, 매개변수, 호출 정보 저장
- 함수가 종료되면 자동으로 해제

#### 2. LIFO 구조
- 빠른 메모리 관리 방식
- 크기가 제한되어 있어 오버플로우 가능

---

## Heap 영역
#### 1. 동적 메모리 할당
- 실행 중 동적으로 할당되는 메모리 영역(JS의 객체, 배열 등)
- 개발자가 해제해야 하는 언어도 있음(C, C++)

#### 2. 큰 메모리 공간
- 속도는 stack보다 느리지만 큰 공간 제공
