## 프로세스의 개념
#### 1. 실행 중인 프로그램 단위
- 메모리에 적재되어 CPU 자원을 할당받아 실행되는 프로그램을 의미
- 독립된 메모리 공간(코드/데이터/스택/힙)을 가짐

#### 2. 자원 관리 단위
- OS가 자원을 할당하고 관리하는 기본 단위
- 프로세스끼리는 독립적이라 메모리 침범 불가 → **IPC** 필요

---

## 스레드의 개념
#### 1. 프로세스 내 실행 흐름 단위
- 프로세스가 할당받은 자원을 이용해 실제로 작업을 수행하는 실행 흐름
- 프로세스 내 에서 **Code, Data, Heap 공유 / Stack만 개별 사용** 

#### 2. 경량 프로세스
- 컨텍스트 스위칭 비용이 프로세스보다 작고 효율적
- 동일 프로세스 내 스레드 간 데이터 공유가 쉬움 → 동기화 문제 주의 필요

---

## 프로세스와 스레드의 차이

| 구분 | 프로세스 | 스레드 |
|---|---|---|
| 정의 | 실행 중인 프로그램 | 프로세스 내 실행 흐름 |
| 메모리 구조 | 독립된 메모리 공간 | Stack만 독립, 나머지 공유 |
| 통신 | IPC 필요 | 메모리 공유로 빠름 |
| 생성 비용 | 크다 | 작다 |
| 장애 영향 | 하나 죽어도 다른 프로세스 영향 X | 하나 문제 → 같은 프로세스 스레드 영향 가능 |
